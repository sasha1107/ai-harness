# Quality Score

Use this file to judge whether a repo still behaves like a good harness.

## Signals Of A Healthy Harness

- Agent entry docs are short and current.
- Architecture rules have matching script checks.
- Plans reflect current work.
- Specs and generated references are versioned.
- Validation is stage-specific and repeatable.

## Drift Indicators

- The important knowledge lives in chat, tickets, or unversioned docs.
- Entry docs turn into encyclopedias.
- New subsystems appear without matching architecture docs.
- Validation scripts stop reflecting actual workflow.
- Completed plans remain active.

## Suggested Cadence

- Review drift indicators at least once per iteration.
- Open cleanup work when the same process failure repeats.
